JavaScript中变量的三种声明方式的区别
发表于 2020-03-04| 更新于 2020-03-28|前端博文原理
字数总计:1.2k|阅读时长: 3 分钟

前言

var、let、const这三个都可以用于JavaScript中变量的声明，var是es5中传统的声明方式，而let和const是es6的新特性。对比这三者的实现原理，大部分人只止步在后两个有块级作用域和变量提升的特点的不同上，少有人知道实现原理。

首先我们来了解下变量的生命周期：

1. 声明（作用域注册一个变量）
2. 初始化（分配内存，初始化为undefined）
3. 赋值

一.声明过程的不同

  • var：遇到有var的作用域，在任何语句执行前都已经完成了声明和初始化，也就是变量提升而且拿到undefined的原因由来
    function： 声明、初始化、赋值一开始就全部完成，所以函数的变量提升优先级更高
  • let：解析器进入一个块级作用域，发现let关键字，变量只是先完成声明，并没有到初始化那一步。此时如果在此作用域提前访问，则报错xx is not defined，这就是暂时性死区的由来。等到解析到有let那一行的时候，才会进入初始化阶段。如果let的那一行是赋值操作，则初始化和赋值同时进行
  • const、class都是同let一样的道理

解析的流程分别是：

  1. 发现作用域有let a，先注册个a，仅仅注册
  2. 没用的第一行
  3. 没用的第二行
  4. a is not defined，程序报错 暂时性死区的表现
  5. 也是注释，没用的第三行
  6. 假设前面那行不报错，a初始化为undefined
  7. a赋值为1

对比于var，let和const只是解耦了声明和初始化的过程，var是在任何语句执行前都已经完成了声明和初始化，let、const仅仅是在任何语句执行前只完成了声明。

二.内存分配上的不同

  • var:直接在栈内存里预分配内存空间，然后等到实际语句执行的时候，再存储对应的变量，如果传的是引用类型，那么会在堆内存里开辟一个内存空间存储实际内容，栈内存会存储一个指向堆内存的指针
  • let:不会在栈内存里预分配内存空间，而且在栈内存分配变量时，做一个检查，如果已经有相同变量名存在就会报错
  • const:不会预分配内存空间，在栈内存分配变量时也会做同样的检查。不过const存储的变量是不可修改的，对于基本类型来说你无法修改定义的值，对于引用类型来说你无法修改栈内存里分配的指针，但是你可以修改指针指向的对象里面的属性

三.顶层对象

ES5中，顶层对象的属性和全局变量是等价的，var 命令和 function 命令声明的全局变量，自然也是顶层对象。

注意：在ES5中，全局变量直接挂载到全局对象的属性上，所以能在window上看到var声明的变量 在ES6中，全局对象的属性和全局变量脱钩，但是为了保持兼容性，旧的不变，所以var、function声明的全局变量依然可以在window对象上看到。var let const三种与之相比，前者因为var的变量会提升到window，但是let个const不会，let ，const会生成块作用域，同一作用域下let和const不能声明同名变量，而var可以。

四.变量提升上的区别

  • let只是创建过程提升，初始化过程并没有提升，所以会产生暂时性死区。
  • var的创建和初始化过程都提升了，所以在赋值前访问会得到undefined.
  • function 的创建、初始化、赋值都被提升了
